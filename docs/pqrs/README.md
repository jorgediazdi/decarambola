# Registro de PQRS

Carpeta para anotar **Peticiones, Quejas, Reclamos y Sugerencias** que no se resuelvan en el acto.

**Autorización del admin:** Las respuestas a PQRS deben ser **autorizadas por el administrador de la plataforma** antes de darse por cerradas. En el registro se debe indicar estado: `pendiente_revision` → `autorizado` (o `rechazado`).

Puedes crear aquí:

- `REGISTRO_PQRS.md` — lista con: fecha, tipo (P/Q/R/S), resumen, involucrado, **estado (pendiente_revision | autorizado | rechazado)**, borrador de respuesta (opcional), fecha de autorización.
- O archivos por año: `2026.md`, etc.

El agente de administración (ver `docs/CONTEXTO_PROYECTO_Y_PQRS.md` y `docs/GEMINI_USO_Y_PQRS.md`) recomienda derivar aquí las PQRS y no dar por cerrada ninguna sin autorización del admin.
